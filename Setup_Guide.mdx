## 🚀 Quick Start Setup

### Prerequisites
- Node.js 18+ installed
- Git installed
- Code editor (VS Code recommended)
- Basic knowledge of React/Next.js (helpful but not required)

### Step 1: Clone and Install

```bash
# Clone the repository
git clone <your-repository-url>
cd naseem_portfolio

# Install dependencies
npm install

# Start development server
npm run dev
```

Visit `http://localhost:3000` to see your portfolio!

### Step 2: Environment Setup

Create a `.env.local` file in the root directory:

```env
# Wisp Blog Configuration
NEXT_PUBLIC_BLOG_ID=your_wisp_blog_id_here

# Optional: Add your own environment variables
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

## 📝 Wisp Blog Setup

Wisp CMS is a headless CMS perfect for developers. Here's how to set it up:

### Step 1: Create Wisp Account

1. Go to [Wisp.blog](https://wisp.blog)
2. Sign up for a free account
3. Create a new blog
4. Note your **Blog ID** from the dashboard

### Step 2: Configure Wisp in Your Project

1. **Add Blog ID to Environment Variables**:
   ```env
   # In .env.local
   NEXT_PUBLIC_BLOG_ID=your_blog_id_here
   ```

2. **Wisp Configuration** (already done in `src/lib/wisp.ts`):
   ```typescript
   import { buildWispClient } from "@wisp-cms/client";

   export const wisp = buildWispClient({
     blogId: process.env.NEXT_PUBLIC_BLOG_ID as string,
   });
   ```

### Step 3: Create Your First Blog Post

1. **In Wisp Dashboard**:
   - Click "New Post"
   - Add a compelling title
   - Write your content using the rich editor
   - Add tags and categories
   - Set featured image
   - Publish when ready

2. **Post Structure**:
   - **Title**: SEO-friendly title
   - **Slug**: URL-friendly version (auto-generated)
   - **Content**: Rich text with images, code blocks, etc.
   - **Excerpt**: Brief description for previews
   - **Featured Image**: Main image for the post
   - **Tags**: For categorization
   - **Status**: Draft or Published

### Step 4: Blog Features

Your blog comes with:
- **Automatic Pagination**: Handled by `BlogPostsPagination` component
- **SEO Optimization**: Meta tags, structured data
- **Responsive Design**: Looks great on all devices
- **Dark Mode Support**: Respects user's theme preference
- **Fast Loading**: Optimized with Next.js Image component

### Step 5: Customize Blog Appearance

**Blog Listing Page** (`src/app/blog/page.tsx`):
- Modify pagination settings
- Update SEO metadata
- Customize post preview layout

**Individual Post Page** (`src/app/blog/[slug]/page.tsx`):
- Customize post layout
- Add sharing buttons
- Modify navigation

## 🎨 Styling & Theming

### Color Customization

**Primary Colors** are defined using TailwindCSS utility classes and CSS custom properties in `src/app/globals.css`:

```css
:root {
  --color-primary: your-primary-color;
  --color-secondary: your-secondary-color;
  --color-accent: your-accent-color;
}

[data-theme="dark"] {
  --color-primary: your-dark-primary-color;
  --color-secondary: your-dark-secondary-color;
  --color-accent: your-dark-accent-color;
}
```

### Typography

**Global Styles** (`src/app/globals.css`):
```css
/* Customize fonts */
body {
  font-family: 'Your-Preferred-Font', sans-serif;
}

h1, h2, h3, h4, h5, h6 {
  font-family: 'Your-Heading-Font', serif;
}
```

### Component Styling

All components use **TailwindCSS** classes. To customize:

1. **Find the component** in `src/components/` or `src/sections/`
2. **Modify Tailwind classes** directly in the JSX
3. **Use responsive prefixes** like `sm:`, `md:`, `lg:` for different screen sizes

Example:
```tsx
// Before
<div className="bg-white dark:bg-gray-900 p-4">

// After (custom colors)
<div className="bg-blue-50 dark:bg-blue-900 p-6">
```

## 🔧 Advanced Customization

### Adding New Sections

1. **Create Component**:
   ```typescript
   // src/sections/new-section.tsx
   export function NewSection() {
     return (
       <section className="py-20">
         <div className="container mx-auto">
           <h2>New Section</h2>
           {/* Your content */}
         </div>
       </section>
     );
   }
   ```

2. **Add to Homepage**:
   ```typescript
   // src/app/page.tsx
   import { NewSection } from "@/sections/new-section";

   export default function Home() {
     return (
       <div className="relative min-h-screen space-y-24">
         {/* Existing sections */}
         <NewSection />
       </div>
     );
   }
   ```

### Custom Components

1. **Create Component**:
   ```typescript
   // src/components/custom-component.tsx
   interface CustomComponentProps {
     title: string;
     description?: string;
   }

   export function CustomComponent({ title, description }: CustomComponentProps) {
     return (
       <div className="p-4 border rounded-lg">
         <h3 className="text-lg font-semibold">{title}</h3>
         {description && <p className="text-gray-600">{description}</p>}
       </div>
     );
   }
   ```

2. **Use Component**:
   ```typescript
   import { CustomComponent } from "@/components/custom-component";

   <CustomComponent title="My Title" description="Optional description" />
   ```

### Adding New Pages

1. **Create Page**:
   ```typescript
   // src/app/about/page.tsx
   ```

2. **Add Navigation** (in `src/components/header.tsx`):

## ❗ Troubleshooting

### Common Issues

#### 1. Blog Posts Not Showing

**Problem**: Blog section is empty or showing errors.

**Solutions**:
```bash
# Check environment variable
echo $NEXT_PUBLIC_BLOG_ID

# Verify Wisp connection
# In browser console, check for network errors
```

**Fix**:
- Ensure `.env.local` has correct `NEXT_PUBLIC_BLOG_ID`
- Verify blog ID in Wisp dashboard
- Check if posts are published (not draft)

#### 2. Build Errors

**Problem**: `npm run build` fails.

**Common Causes**:
- TypeScript errors
- Missing environment variables
- Import/export issues

**Solutions**:
```bash
# Check TypeScript errors
npm run lint

# Build with verbose output
npm run build -- --debug
```

#### 3. Styling Issues

**Problem**: Components look broken or unstyled.

**Solutions**:
- Check Tailwind CSS configuration
- Verify CSS imports in `globals.css`
- Clear browser cache
- Check for CSS class conflicts

#### 4. Images Not Loading

**Problem**: Images in siteConfig.ts not displaying.

**Solutions**:
- Use absolute URLs for external images
- For local images, place in `public/` folder
- Check image URLs are accessible
- Verify Next.js Image component usage

### Performance Issues

#### Slow Loading

**Optimizations**:
```typescript
// Optimize images in siteConfig.ts
image: {
  src: "optimized-image-url",
  alt: "descriptive-alt-text",
  width: 330,  // Specify dimensions
  height: 160
}
```

#### Bundle Size

**Check bundle size**:
```bash
npm run build
# Check .next/static/ folder sizes
```

**Reduce bundle**:
- Remove unused dependencies
- Use dynamic imports for large components
- Optimize images

### Getting Help

1. **Check Console**: Browser DevTools → Console
2. **Check Network**: DevTools → Network tab
3. **GitHub Issues**: Create issue with:
   - Error message
   - Steps to reproduce
   - Environment details
